My Experiences

Web Development Java C Unity Others

Web Development was my first contact with programming languages. I first saw it with my brother because he had computer science in his high school diploma. He programmed at home because there was homework's, for example, so he showed me some things like HTML or CSS. After I showed interest, I also found other forms of web development like PHP or JS(JavaScript). But I fell in love with PHP and I hated JS a bit. Because I loved PHP, I found again new ways again, and found the framework Symfony. Symfony is a framework for everything like shopping sites or social media sites. But after a while I got bored with web development and found other programming languages.

Java was my second programming language. I got to know it through Minecraft and then I also programmed e.g. plugin. Meanwhile I saw that you can do other things with it like JFrames (applications) or even small games with LibGDX (library). But I also kept coming back to smaller gadgets like a cmd- (Windows command prompt) calculator. Because of the urge to develop games which I made with LibGDX at that time I also came to the game engine Unity.

I was also interested in lowlevel languages and I started to get more and more involved in OS development. So I first came to assembler which is in my opinion very difficult to understand and doesn't take much work off your shoulders with e.g. simple loops. Therefore I came to the language C. I had to learn it first, but Java gave me most of the basics. In C you had all the advantages of a high level programming language including an easier understanding of the code and a lot of access to the hardware. I only have contact with C because of my kernel (CCOS).

I started with Unity because I was very interested in game development. First I programmed smaller games to get into the C# language. I programmed both 2D and 3D games but unfortunately most of them were not finished because other things or ideas of relationships kept me from finishing them. Now I am working on a small 2D mobile dungeon crawler. I am also working in the team MoveByte on a 3D game for the desktop.

In this learning and development process one had of course also contact with other engines or programming languages. That's also how I got to know the Unreal engine, for example. But it was too confusing for me and somehow it didn't fit me so well, so I stayed with unity. I also had contact to the programming language C++ but it didn't fit my taste so I stayed with the other programming languages.
